### Primary Key

Every entry type has 1 primary key. The primary key is a text field, the value may be calculated. When the primary key is not calculated its value can be updated like any other text field. However, when requesting the value of the primary key it is always presented as a complex object.

**Schema: Primary Key**

```json
{
  "apiName": "Name",
  "fieldType": 1,
  "isRequired": true,
  "allowDuplicates": false,
  "warnOnNearDuplicates": false,
  "isMoney": false,
  "isMultiSelect": false,
  "entryLists": [],
  "systemFieldType": 9,
  "isKey": false,
  "isCalculated": false,
  "isAttachment": false,
  "isStoreRequestSupported": true,
  "id": 22247,
  "name": "Name",
  "entryListId": 22246
}
```

### Text

The most common field type, used to store string values. There are several sub-types of text fields (single line, multi-line, URL, rich-text, phone). The sub-type affects how the field is displayed in the UI and does not change the field JSON properties. Strings are stored as HTML encoded values. If you submit a string value it will be HTML encoded before saving. When searching for a string value we escaped. You should HTML encode values before submitting, if not values will be automatically encoded before saving. Returned string values are HTML encoded and should be decoded to get the actual string values.

**Schema: Text Field**

```json
{
  "apiName": "TextEmail",
  "fieldType": 1,
  "isRequired": false,
  "allowDuplicates": true,
  "warnOnNearDuplicates": false,
  "isMoney": false,
  "isMultiSelect": false,
  "entryLists": [],
  "systemFieldType": 0,
  "isKey": false,
  "isCalculated": false,
  "isAttachment": false,
  "isStoreRequestSupported": true,
  "id": 22255,
  "name": "Text Email",
  "entryListId": 22246
}
```

```json
{
  "entryId": 3079558,
```

```json
  "value": "Test@email.com",
  "ignoreNearDups": true,
  "entryId": 2034,
```

GET | POST | PUT
On a POST request the entryId would be a negative number while the rowId will be the actual unique EntryId. On PUT calls both rowId and entryId are the same. rowId is not included on requests where the data is not modified.

```json
{
  "entryId": 3079558,
  "fieldId": 22255,
  "rowId": 3079558,

}
```

Texts fields always return a flat string value except when the text field is the primary key for the entryType. View Section on Primary Key to see how primary keys are returned.

```json

  "EntryId": 3079558,
  "TextEmail": "Test@email.com"
}
```

### Choice

Choice fields allows you to create pick lists. Choice fields can be chained together to create multi-level dependencies. Choice fields can be single select or multi-select. All available options for a choice field is displayed in the field schema. You're able to append new choice options via the Schema API. When updating or modifying a choice value you must pass the ID of the choice option, when passing multiple values use an array of integers.

**Schema: Choice Field**

```json
{
  "apiName": "ChoiceLevel1",
  "fieldType": 2,
  "isRequired": false,
  "allowDuplicates": true,
  "warnOnNearDuplicates": false,
  "isMoney": false,
  "isMultiSelect": true,
  "choiceFieldId": 3347,
  "choiceOrder": 1,
  "entryLists": [],
  "systemFieldType": 0,
  "choiceValues": [
    {
      "seqNumber": 1,
      "id": 3079556,
      "name": "One",
      "entryListId": -6
    },
    {
      "seqNumber": 2,
      "id": 3079555,
      "name": "Two",
      "entryListId": -6
    }
  ]
}
```

### Binary

Binary fields exists only on two entry types (Attachment and Document). When you create an attachment field on any other entry type (ie. contact, company, email, etc..) the attachment field is represented as a reference field to the attachment list.

When attempting to upload a binary object to an attachment field:

1. Upload to Attachment List
2. Capture Entry Id of uploaded object(s)
3. Pass Entry Id as reference to the Attachment Reference field on the record that owns the attachment.

| Name  | Description                                                                                          |
| ----- | ---------------------------------------------------- |

| Document | This is the actual binary blob |
| FileType | This is the binary extension without the `.` |
| Size | This is optional, can be used for hash comparison if provided on post/patch. Will be automatically filled in on successful upload. |
